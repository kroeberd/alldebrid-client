Features

Everything you need.
Nothing you don't.

Built for self-hosters who want reliable, hands-off torrent automation with full visibility into what's happening.

🖥️
Web UI
Clean single-page dashboard: all torrents, status, progress, events, file list. Dark tech aesthetic, mobile-friendly.
📁
Watch folder
Drop a .torrent or .magnet file and it's automatically imported. Processed files are moved to a separate folder.
🔗
Sonarr / Radarr
Direct integration as a download client. After completion, Sonarr/Radarr is triggered to import the file automatically.
🌐
REST API
Full REST API for adding magnets, querying status, triggering syncs and webhooks. Swagger UI at /api/docs.
⬇️
aria2 integration
Slot-based queue with configurable concurrency. Existing downloads detected on restart — no duplicates, no orphans.
🔔
Discord webhooks
Rich embeds for every event. Separate channel support for added vs completed. Custom avatar and username.
🌿
FlexGet v3
Schedule and trigger FlexGet tasks from the UI. Per-event Discord notifications for each task result.
🚫
File filters
Block by extension, keyword or minimum size. All-blocked torrents auto-removed from AllDebrid. Partial downloads continue normally.
🗄️
SQLite or PostgreSQL
SQLite by default — zero config. Switch to external PostgreSQL via environment variable for multi-container setups.
📊
Statistics
Rolling snapshots with configurable retention. Periodic summary reports via webhook.
💾
Automatic backups
SQLite backups on a configurable schedule. Retained by number of days, stored in the data folder.
🔧
Config validation
Settings validated and auto-corrected on every startup. Invalid URLs cleared, numeric fields clamped, stale JSON reset.
Get started

Up in 60 seconds.

One compose file. Configure the rest in the browser.

1
Copy the compose file

Adjust volume paths for your setup. The download folder must match your aria2 download path.

2
Open the Web UI

http://your-server:8080 → Settings → enter your AllDebrid API key and aria2 RPC URL → Save.

3
Add your first magnet

Paste a magnet link, drop a .magnet file in the watch folder, or configure Sonarr/Radarr.

4
Unraid App Store

Search for AllDebrid-Client in Community Apps — all paths are pre-filled.

docker-compose.yml 
services:
  alldebrid-client:
    image: kroeberd/alldebrid-client:latest
    container_name: alldebrid-client
    restart: unless-stopped
    network_mode: host
    environment:
      - TZ=Europe/Berlin
      - LOG_LEVEL=INFO
    volumes:
      - /mnt/appdata/alldebrid/config:/app/config
      - /mnt/appdata/alldebrid/data:/app/data
      - /mnt/data/torrents/watch:/app/data/watch
      - /mnt/data/torrents/processed:/app/data/processed
      - /mnt/downloads/alldebrid:/download
docker run (one-liner) 
docker run -d \
  --name alldebrid-client \
  --restart unless-stopped \
  --network host \
  -e TZ=Europe/Berlin \
  -v /mnt/appdata/alldebrid/config:/app/config \
  -v /mnt/appdata/alldebrid/data:/app/data \
  -v /mnt/downloads/alldebrid:/download \
  kroeberd/alldebrid-client:latest

Why AllDebrid-Client

Built for reliability.

Designed to run unattended for weeks. Every edge case is handled.

What it handles automatically

  • Duplicate download prevention across restarts
  • Stuck download detection and cleanup
  • aria2 GID tracking — no orphaned jobs on restart
  • AllDebrid magnet auto-deleted after completion
  • All-filtered torrents → complete, not error
  • Config auto-corrected at startup
  • Rate-limit aware AllDebrid API calls
  • FlexGet overlap guard — no concurrent runs

Stack

  • Python 3.12 + FastAPI — fully async
  • Single-file vanilla JS frontend (no build step)
  • aiohttp, aiosqlite, asyncpg
  • SQLite default, PostgreSQL optional
  • Docker image under 200 MB, single container
  • MIT licensed, 100 % open source
  • 109+ automated tests in CI
