Five steps to your first download

Complete the steps below once and you are ready to go.

1
Get your AllDebrid API key
Open alldebrid.com/apikeys, create a key called AllDebrid-Client and copy it. An active AllDebrid subscription is required — the free tier does not support the download API.
2
Enter the key in Settings → Integrations
Click Settings in the sidebar → 🔗 Integrations tab → paste the key into API Key → Save Settings. The orange dot in the header should turn green within a few seconds.
3
Set the download folder
Settings → ⚡ General → Download Folder. This must be the path inside the container (e.g. /download). On Docker / Unraid map your host folder to this path via a volume binding. Example Unraid: host path /mnt/user/Downloads → container path /download.
4
Choose your aria2 setup
Settings → ⬇️ Download → aria2 Mode.
Built-in (recommended): aria2 is managed automatically — nothing else to do.
External: point the RPC URL to your own aria2 instance and enter its secret.
5
Add your first torrent
Click + Add Torrent in the top bar and paste a magnet link or upload a .torrent file. The torrent is sent to AllDebrid, cached on their servers, then delivered to your disk via aria2 — usually within seconds for popular content.
🐳 Docker Compose quick-reference
services:
  alldebrid-client:
    image: kroeberd/alldebrid-client:latest
    container_name: alldebrid-client
    restart: unless-stopped
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data          # settings & database
      - /your/host/downloads:/download  # download target
    environment:
      - PUID=1000   # match your host user
      - PGID=1000

The download pipeline

AllDebrid-Client is a bridge between torrent sources and your storage. Every download goes through five stages:

1
Upload
The magnet or .torrent is sent to AllDebrid's servers. They download the actual torrent — often from a shared cache, which means the files may already be available in seconds.
2
Poll
AllDebrid-Client checks AllDebrid every 30 s (configurable) for the torrent status. When AllDebrid reports it as ready, the app moves to the next step.
3
Unlock
Each file link is "unlocked" — AllDebrid generates a time-limited direct HTTPS download URL. Up to 5 unlock calls run in parallel (rate-limited to avoid 503 errors).
4
Download
The URL is queued in aria2. aria2 downloads the file to your disk using multiple parallel connections (configurable). The Downloads view shows live progress.
5
Notify & Import
A Discord notification is sent (if configured). If Sonarr or Radarr integration is enabled, the import API is called automatically.

Torrent status explained

QueuedUploaded to AllDebrid, waiting for them to process it. Normal for new or rare torrents.
DownloadingAllDebrid is downloading the torrent to their servers. Popular content is usually instant.
DispatchedFiles have been unlocked and handed to aria2 for download to your disk.
CompletedAll files saved to your download folder. Sonarr/Radarr import triggered if configured.
ErrorSomething failed. Check the Event Log for details. The torrent can be retried manually.

aria2 — the download engine

Built-in vs External

Built-in ✓ Default and recommended. aria2 is started, stopped, and configured automatically by the app. No extra containers, no manual config, no port forwarding needed.
External Use if you already run aria2 separately (e.g. in another container or on a different machine). Set the RPC URL and secret. If aria2 is in a different container, use the container name as hostname and ensure both share the same Docker network. Set Download Path Override if the path inside the aria2 container differs from the one in this container.

Key settings

Simultaneous DownloadsHow many files aria2 downloads at once. Default: 3. Increasing this uses more RAM; 5 is a practical ceiling for most setups.
Split / ConnectionsNumber of parallel segments per file. Default: 4. More segments = faster on fast servers but more RAM. Keep Max Connections per Server ≤ Split.
Disk Cache0 = disabled (best for local SSD, ~4 MB RAM). 16M = recommended for network mounts (NFS, SMB, mergerfs) — batches writes, reduces FUSE round-trips, can actually lower peak RAM.
File Allocationnone (recommended): no pre-allocation, downloads start immediately. falloc: fast on modern filesystems (ext4, XFS, BTRFS) but blocks aria2 completely until allocation finishes — avoid for large files on network shares. prealloc: slower, works everywhere.
Speed LimitMinimum speed before a connection is dropped. Use K/M suffix (e.g. 100K). 0 = no limit.
Memory optimisations applied automatically by the built-in mode
MALLOC_ARENA_MAX=1 · MALLOC_TRIM_THRESHOLD_=65536 · --async-dns=false · --no-netrc=true · --file-allocation=none · session file cleared on restart so the DB is the single source of truth.

Understanding RAM usage

20 GB "used RAM" during downloads is almost always the kernel page cache — not a memory leak. The kernel caches every written byte in RAM and releases it only when another process needs it. On a dedicated server this may never happen, so it grows with every download session.

Three separate RAM sources

aria2 process heap With disk-cache=0: ~4–50 MB. This is real process RAM. The built-in mode applies MALLOC_ARENA_MAX=1 to slow arena growth.
Kernel page cache ★ Every downloaded byte is cached in RAM by the Linux kernel. Can reach 10–30 GB. Shows as "used" in Unraid, htop, and Docker stats but is reclaimed automatically when another process needs memory. Not a leak.
glibc malloc arenas After many download cycles glibc retains freed pages in internal arenas. Grows slowly. Mitigated by MALLOC_ARENA_MAX=1. Fully reset by restarting the aria2 process.

How to check

In Settings → ⬇️ Download there are three buttons:

  • Memory Info — reads /proc/meminfo and shows really_used (process RAM) vs page_cache (kernel file cache) vs available. If available is large, you have no problem.
  • Drop Page Cache — calls posix_fadvise(DONTNEED) on all completed files, telling the kernel to release their cached pages immediately. Safe on any Linux filesystem; a no-op on Windows/macOS.
  • Run Cleanup — removes stopped download results from aria2's internal result list.
Network mounts (NFS, SMB, mergerfs on Unraid)
Set Disk Cache to 16M in the Download settings. Without a cache, aria2 sends every small HTTP chunk directly to the FUSE filesystem, causing many round-trips and higher peak RAM. A 16 M buffer coalesces writes and can actually reduce RSS compared to 0.

Integrations

📺 Sonarr / 🎬 Radarr

After a torrent finishes downloading, AllDebrid-Client calls the Sonarr or Radarr import API so your library is updated automatically.

  1. Enable the integration in Settings → 🔗 Integrations.
  2. Enter the base URL (e.g. http://sonarr:8989) and the API key from Sonarr/Radarr → Settings → General.
  3. Ensure PUID/PGID on this container matches the user that Sonarr/Radarr runs as, so both can read the downloaded files.
  4. Use the Test button to verify the connection before saving.

🔔 Discord Webhooks

Create a webhook in your Discord server (Server Settings → Integrations → Webhooks), copy the URL, and paste it in Settings → 🔔 Discord. You can use a separate webhook URL for "torrent added" events to route notifications to a different channel.

🔍 Jackett

Jackett lets you search hundreds of torrent trackers from the built-in Search view. Enable it in Settings → 🔍 Jackett, enter the base URL and API key (found in the Jackett dashboard top-right). The Search sidebar entry appears once Jackett is enabled and connected. The API key is proxied through the backend — it is never sent to the browser.

📁 Watch Folder

Drop .torrent or .magnet files into the configured watch folder. AllDebrid-Client scans it every 10 seconds and imports them automatically. After import the file is deleted, or moved to the Processed Folder if one is configured. Point Sonarr, Radarr, RSS tools, or scripts at the watch folder for fully automated workflows.

🤖 FlexGet

AllDebrid-Client can trigger and monitor FlexGet tasks from the UI. Enable in Settings → 🤖 FlexGet, enter the URL and run flexget web gentoken to get the API key. Tasks appear in the FlexGet sidebar view where you can run them on demand or view recent execution history.

Settings reference

Settings are organised into five tabs. All changes take effect after clicking Save Settings.

⚡ General

AllDebrid API KeyRequired. Get from alldebrid.com/apikeys.
Download FolderPath inside the container where aria2 saves files. Must match your Docker volume mapping.
Watch / Processed FolderOptional. Drop .torrent/.magnet files in Watch for automatic import.
Max Concurrent DownloadsHow many AllDebrid magnets are processed at once. Default: 3.
Stuck Timeout (h)Hours before a stuck torrent is auto-retried. 0 = disabled.
Rate Limit / SyncAllDebrid API calls per minute (default 60) and background reconciliation interval.

⬇️ Download

aria2 ModeBuilt-in (default, no setup) or External (point to your own aria2 RPC endpoint).
Simultaneous DownloadsFiles aria2 downloads at once. Default 3. Recommended max 5.
Disk Cache0 for local SSD; 16M for network mounts (NFS/SMB/mergerfs). See RAM & Memory tab.
File Allocationnone (recommended). falloc blocks aria2 during pre-allocation — avoid for large files on network shares.

🔔 Notifications

Webhook URLMain Discord webhook for all events.
Added WebhookOptional separate webhook for "torrent added" events (different channel).
Event togglesEnable/disable notifications per event type: Added, Completed, Error.

🔌 Services

Sonarr / RadarrBase URL + API key. Triggers import scan after download completes.
JackettURL + API key. Enables the Search view with multi-indexer chip picker.
FlexGetURL + token (run flexget web gentoken). Tasks shown in sidebar.
LabelsOne label per line — shown in the Add Torrent dialog and filter chips.

🛠️ Advanced

File FiltersBlock by extension, keyword, or minimum size (MB). Filtered files are skipped before download.
PollingAllDebrid check interval (default 30 s) and watch folder scan interval (default 10 s).
BackupsAutomatic SQLite backups on a configurable schedule with retention days.
ReportingPeriodic statistics summary sent to a Discord webhook. Configure interval and window.
DatabaseSQLite (default) or PostgreSQL. Migration tool included. Wipe button is guarded by a toggle.

Troubleshooting

Torrent stuck in Queued for a long time

AllDebrid is downloading the torrent to their servers. This is normal for unpopular or brand-new content. Popular content is usually cached instantly. If stuck >30 min, remove and re-add the magnet. The Stuck Timeout setting (Settings → Integrations) will auto-retry after the configured hours.

AllDebrid returns HTTP 503

Rate limit hit during the unlock step. AllDebrid-Client caps parallel unlock calls at 5 and adds retry logic, but very large torrents with hundreds of files can still trigger this. Reduce Simultaneous Downloads in Settings → Download, or lower the Rate Limit setting.

Files download but Sonarr / Radarr doesn't import them

Check: (1) The download folder path matches what Sonarr/Radarr expects — both containers must see the files at the same host path. (2) PUID/PGID is set correctly so both containers share file ownership. (3) The API key is correct — use the Test button in Settings. (4) The Sonarr/Radarr root folder is configured and the episode/movie is monitored.

High RAM usage (20 GB+)

Almost certainly the Linux kernel page cache, not a leak. See the RAM & Memory tab for a full explanation. Use Memory Info in Settings → Download to confirm — if available is large, you have no problem. Click Drop Page Cache to release it immediately. If your download folder is on a network mount, set Disk Cache to 16M.

External aria2 not connecting

Check: (1) aria2 is started with --enable-rpc=true --rpc-listen-all=true. (2) The RPC URL in settings uses the correct hostname and port (default 6800). (3) The secret matches (or is empty if no secret is set). (4) If aria2 is in a separate Docker container, both must be on the same Docker network and you should use the container name as hostname.

Downloaded files have wrong ownership / permissions

Set PUID and PGID environment variables to match your host user (find them with id on the host). The container's entrypoint will run aria2 as that user so all created files have the correct ownership.

Remux / large files not downloading

Usually a Sonarr/Radarr quality profile issue, not AllDebrid-Client. Check: (1) The quality profile allows the file size (Sonarr/Radarr have per-quality size limits). (2) The indexer settings allow the file. AllDebrid-Client passes through whatever AllDebrid provides — it does not filter by size except via the Filters settings.

Database locked / SQLite errors

Restart the container. If the error persists, check that no other process has the database file open. The database file is at /app/data/alldebrid.db inside the container. If corruption is suspected, use Settings → 🗄️ Database → Create Backup before attempting any repairs.

Settings are applied immediately after saving.
