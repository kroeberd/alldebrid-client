❓ Help & Documentation

🚀 Quick Start — 4 steps

1
AllDebrid API key — Settings → ⚡ General. Get it at alldebrid.com/apikeys.
2
Download folder — Settings → General → Folders. Set the path inside the container (e.g. /download). Map your host folder to this path in Docker.
3
aria2 (built-in, default) — Nothing to do. The built-in aria2 starts automatically. To use an external aria2, go to Settings → ⬇️ Download and switch the mode.
4
Add a torrent — Click + Add Torrent in the top bar and paste a magnet link. Done.

📖 How it works

AllDebrid-Client is a pipeline between torrent sources and your storage:

1. UploadMagnet is sent to AllDebrid. They download the torrent (often instantly from cache).
2. PollAllDebrid-Client checks AllDebrid every 30 s until ready.
3. UnlockEach file gets a temporary HTTPS download URL from AllDebrid (max 5 parallel).
4. DownloadURL is handed to aria2 which downloads the file using multiple parallel connections.
5. NotifyDiscord notification sent when complete (if configured).
QueuedUploaded to AllDebrid, waiting for them to process.
DownloadingAllDebrid is downloading the torrent to their servers.
DispatchedFiles are being downloaded by aria2 to your disk.
CompletedAll files saved to your download folder.
ErrorSomething failed. Check the event log or retry.

⬇️ aria2 — Built-in vs External

Built-in ✓Default. Starts with the app, uses the same download folder, no extra config. Recommended for most users.
ExternalUse if you already run aria2 separately. Set the RPC URL and secret. If aria2 is in another Docker container, use the container name as hostname and set Download Path Override if paths differ.

Key settings

Simultaneous DownloadsFiles downloaded at once. Default: 3. More = more RAM.
Connections per FileParallel segments per file. Default: 4. Higher = faster but more RAM.
Disk Cache0 = best for local SSD. 16M = better for NFS / SMB / mergerfs. Batches writes to reduce filesystem calls.
File Allocationnone = recommended (instant start). falloc = fast on modern FS but blocks aria2 while pre-allocating — avoid for large files on network shares.

🧠 Understanding RAM usage

20 GB RAM during downloads is usually the kernel page cache — not a memory leak.
aria2 heap~10–50 MB with disk-cache=0. Real process RAM.
Page cache ★Every downloaded byte is cached in RAM by the kernel. Can reach tens of GB. Shows as "used" in Unraid/htop/Docker stats — but is reclaimed automatically when needed. Not a problem.
glibc arenasAfter many download cycles, glibc retains freed pages. Grows slowly. Mitigated by MALLOC_ARENA_MAX=1 (set automatically). Cured by restarting aria2.

How to check: Settings → Download → Memory & Diagnostics → 📊 Memory Info.
If available RAM is large and page_cache explains the "used" number — everything is fine.
Click 🗑️ Drop Page Cache to release cached pages for completed files immediately (Linux only, safe).

Network mounts (NFS / SMB / mergerfs): Set Disk Cache to 16M. Batching writes reduces FUSE round-trips and can actually lower peak RAM compared to 0.

🔌 Integrations

Sonarr / Radarr — Enable in Settings → Services. After a download completes, AllDebrid-Client calls the import API automatically. PUID/PGID must match so both containers can access the files.
Jackett — Enable in Settings → Services → Jackett. The Search sidebar entry appears. Search any configured tracker and add results directly to your queue.
Discord — Create a webhook (Server Settings → Integrations → Webhooks) and paste in Settings → Notifications. Separate webhooks can be set for "added" events and for Jackett/FlexGet.
Watch Folder — Drop .torrent or .magnet files in the configured folder. Picked up every 10 s and imported automatically. Files are deleted or moved to the Processed Folder after import.
FlexGet — Enable in Settings → Services. Schedule and trigger FlexGet tasks from the UI. Run flexget web gentoken to get the API key.

🔧 Troubleshooting

Torrent stuck in Queued
AllDebrid is downloading the torrent. Normal for unpopular content. The stuck-download timeout (Settings → General) auto-retries after the configured time.
HTTP 503 errors from AllDebrid
Rate limit hit. AllDebrid-Client caps unlock calls at 5 concurrent. If frequent, reduce Simultaneous Downloads in Settings → Download.
Settings page is blank / white screen
JavaScript error in the settings renderer. Open browser console (F12 → Console) for the error message. Try a hard refresh (Ctrl+Shift+R). Check container logs if it persists.
Sonarr / Radarr not importing
Check: (1) download folder path matches what Sonarr/Radarr expects, (2) PUID/PGID matches both containers, (3) API key is correct — use Test Connection button.
High RAM (20 GB+)
Almost certainly the kernel page cache, not a leak. See the RAM tab for details. Click Memory Info in Settings → Download to confirm. Set Disk Cache to 16M if using a network share.
External aria2 not connecting
Check: (1) aria2 runs with --enable-rpc, (2) RPC URL/port is correct (default 6800), (3) secret matches, (4) for Docker: use the container name as host and ensure both are on the same network.
